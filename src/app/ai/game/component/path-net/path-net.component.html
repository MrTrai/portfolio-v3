Path Network Navigation

This section will introduce on how to generate a path network for the AI agent to perform path finding. Source Code

What is a path network?

A path network is a static network of path edges that allow the agent to move to any location in the game.

Generate the path network from a nav mesh

Algorithm

Approach 1: Centroid

  1. For each of the mesh in the nav mesh, find the centroid
  2. Connect each of the centroid to all the neighbors such that the path edge doesn't cut another object with consideration of the agent's radius.

Approach 2: Edge midpoint

  1. For each of the line of each mesh in the nav mesh, find the midpoint
  2. Connect each of the midpoint to all the neighbors such that the path edge doesn't cut another object with consideration of the agent's radius.

Note: From personal tests, I find the midpoint approach is better at producing a path net that can cover the whole map.

How is path network used?

  1. In order for the agent to move from point A to point B, it first finds the 2 points nearest to A and B from the path network, says C and D respectively.
  2. Move the agent from A to C
  3. The agent now performs a standard A* path finding to D given the network edges.
  4. It then follows the path to point D and finally moves to B.

The main goal of path network is to reduce the search space for path finding. In games where performance is crucial, optimizing path finding run time is more important than ever.
